# Main
## Авторизация студента
После того, как студент первый раз вошёл а приложение (для нас он зарегистрировался), нужно создать в базе данных запись с его логином (email, который мы получили от авторизации), номером группы и уникальным uuid этого пользователя. Также нужно хранить информацию о дате-времени регистрации и о дате-времени последней активности для аналитики

## Расписание
1. Расписание нужно кешировать на нашей стороне на 2 недели вперёд. В случае, если оригинальное API ruz недоступно, мы достаём расписание из кеша и показываем его пользователю, добавив плашку о том, что это офлайн-данные. Кешируем расписание только для тех учебных групп, студенты которых зарегистрированы в нашей системе
2. [ПОКА НЕ ДЕЛАТЬ!] Нужно отслеживать изменения в расписании: изменение преподавателей, аудитории, появления нового занятия, пропадания старого или перемещения занятия на другое время (если препод, и дисциплина полностью пропали из одного слота и появились в другом). Для этого используем кэш, который описан в п.1
3. В карточках занятий нужно показывать анимацию мигающего красного кружочка, если сейчас идёт данная дисциплина (кружок помещаем над временем начала занятия)
4. Бадж с типом занятия нужно перенести выше названия занятия.
5. Скролл влево-вправо для переключения между днями работает только если свайпить по карточке занятия, а если по пустому пространству - нет, нужно пофиксить.
6. Пользователю нужно дать возможность добавлять в избранное расписание других групп или преподавателей. Для этого нужно в нижней части экрана расписания над навбаром добавить панельку, где по центру будет написана текущая группа или ФИО преподавателя, чьё расписания смотрится (аналогично тому, как сверху показывается дата текущего дня). В правой части этой панель нужно добавить кнопку-шеврон вверх, при нажатии на которую студент сможет выбрать другую группу/препода. Добавление новых групп и преподов нужно сделать через строку поиска - пользователь вводит номер группы или фио препода (например 5130904/20102 или Петров Александр), после чего через апишку ищем групп, преподов, показываем, пользователь выбирает элемент и он сохраняется как избранный в его список. Список избранного храним в БД.
